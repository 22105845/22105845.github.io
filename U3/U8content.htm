U8 Content

What it was made for:

My tutor asked me to design a prototype interactive app that will help children learn the alphabet.

The prototype must contain at least six letters of the alphabet to demonstrate how the app for CambriaTek will work. After further discussions with CambriaTek mt tutor made the following observations.

  • The users of the App will be between 3 & 7 years old, so the user interface should be appealing to this age range and be simple to use.
  • The app will need to help the users recognise the shapes and sounds of letters of the alphabet.
  • There will be no requirement for achievements to be graded or recorded as the client wants the app to be fun to use without the risk of the children getting demotivated by negative feedback.

How it was developed:

The development process included creating a multi-screen interface that reflects the planned interface, with assets on each screen. The app is also be multi-functional.

  • Prepare and gather your pre-defined code and ready-made assets, such as buttons and images. All pre-defined code and assets should be fully listed in a fully-referenced source table.
  • Make sure that your assets demonstrate awareness of user requirements and the intended purpose of the app; for example, bitmap images should be optimised.
  • Creating an interface which demonstrates an awareness of the user requirements and intended purpose of the app by assigning code to assets, e.g. buttons to control behaviour.
  • Edit pre-defined code and develop some original code to fulfil the design requirements of the app.
  • Write comments within the code to explain how it works.
